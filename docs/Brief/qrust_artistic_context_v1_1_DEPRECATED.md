## Convention pour les futurs projets Qrust

Tout nouveau projet musical Qrust suit le pattern :

1. Hériter automatiquement de `qrust_professional_context.md` (le socle invariant)
2. Créer un nouveau `project_<nom_projet>_context.md` pour ses paramètres spécifiques

Exemples futurs possibles :
- `project_<future_track>_context.md` (peut être un autre genre, autres targets)
- `project_<remix_collab>_context.md` (pour un remix ou collab future)
